# VM

## Execution model

The VM executes a token stream in reading order. Each token is executed as:

```
Select → Bound → Seal
```

### Shin three-point attachment execution

`ש` follows the standard letter contract with direction-parameterized attachment.

- **Select:** current focus (`F`) - the handle forwarded by the preceding letter.

- **Bound** (parameterized by direction):
  - **External / Tripod** (`שׁ`, default undotted): allocate three outward-facing ports from `F` and add `cont(F, p1)`, `cont(F, p2)`, `cont(F, p3)`. Do not add edges between `p1`, `p2`, `p3`.
  - **Internal / Triangle** (`שׂ`): allocate three compartments and add `sub(F, c1)`, `sub(F, c2)`, `sub(F, c3)`, plus `sub(c1, c2)`, `sub(c2, c3)`, `sub(c3, c1)` as a closed loop.
  - The triangle's closed loop yields rigidity as a topological fact, not a metadata flag.

- **Seal:** focus remains `F`; thread forwards `F`. All three points remain live; there is no active branch selection.

Dot selection:

- Right dot -> `direction=external` (tripod)
- Left dot -> `direction=internal` (triangle)
- Undotted -> `direction=external`

Milui:

- `שי״ן = three-point attachment (ש) -> pin (י) -> straighten (ן)`.
- After tripling the attachment surface, commit to one pinned line and lock it as continuation.

### Toch dot resolution

The Unicode inside-dot mark `U+05BC` MUST be resolved by host-letter context before execution:

- If the token contains `U+05BC` and base letter is `ה`, set `dot_kind=mappiq`.
- Else if the token contains `U+05BC`, base letter is `ו`, and no other Sof vowel marks are present, set `dot_kind=shuruk`.
- Else if the token contains `U+05BC`, set `dot_kind=dagesh`.

Execution rules tied to this resolution:

- `dagesh` applies hardening behavior.
- `mappiq` does not harden; it does not force any special `ה` execution mode.
- `shuruk` does not harden; it activates ו carrier mode.

Current `ה` execution is the resolved head-with-leg form:

- select current construct `X` (or word-entry ambient `W₀` if none exists),
- allocate head `h` and add `head_of(h, X)`, `carry(X, h)`, `supp(h, X)`,
- allocate detached leg `ℓ` and add `cont(h, ℓ)`, `carry(h, ℓ)`, `supp(ℓ, h)`,
- seal with `F := h` and export `ℓ` as an adjunct of `h`.

### Registers

Minimum required registers:

- `Ω`: ambient/root handle (always defined).
- `F`: current focus handle.
- `R`: last-residue handle (defaults to `⊥`).
- `K`: handle stack (LIFO).
- `E`: environment stack (scoping frames).
- `W`: watchlist or persistent selectors (optional but recommended).
- `segment`: segment context `{ segmentId, OStack }`.
- `OStack_word`: active obligation stack (LIFO), aliased to `segment.OStack`.
- `A`: phrase accumulator (`WordValue[]`).
- `activeConstruct`: current word baseline construct (`C0`) allocated at `WordStart`.
- `aliasEdges`: symmetric alias adjacency (for `Alias(F0, C0)` and traversal/debug).
- `τ`: step counter.
- `H`: event log (optional but recommended).

Initial state (before first token):

```
Ω := root scope handle
F := Ω
R := ⊥
K := [F, R]
W := []
segment := { segmentId: 0, OStack: [] }
OStack_word := segment.OStack
A := []
activeConstruct := unset
aliasEdges := []
τ := 0
```

## Operand selection policy

Each letter declares:

- `arity_req`, `arity_opt`
- `distinct_required`, `distinct_optional`
- `reflexive_ok`
- input/output handle kinds

Required operands are sourced **in order** from:

1. `K` (stack)
2. `W` (watchlist)
3. `F` (focus)
4. `R` (residue)
5. `Ω` (ambient)

Optional operands are supplied only if available without forcing duplicates.

### Underflow policy

If required operands are missing:

- If `reflexive_ok` is true, duplicate the last selected operand.
- Otherwise, fill with `⊥`.

If any filled operand violates the operator’s type signature, the program is ill-formed (compile error).

## Seal policy

`Seal` MUST:

- push `h_out` onto `K`
- set `F := h_out` in the current frame
- set `R := r_out`

`r_out` is total: every letter yields a residue; if none, use `⊥`.

`Seal` MAY allocate fresh, typed handles. Handle IDs MUST be deterministic; see `spec/70-DETERMINISM.md` for ID generation requirements.

## Internal vs external effects

Each operator has:

- an **internal act** (what it constructs/changes inside the focus), and
- an **external residue** (what becomes “outside” as a necessary side-effect).

Residues are explicit via `R`, and external effects are recorded in boundaries and the event log `H` where applicable.

## Two-level algebra (words, phrases)

A **word** is the ordered composition of its letters:

```
W(word) = ℓ1 ⊗ ℓ2 ⊗ ... ⊗ ℓn
```

Where `⊗` is non-commutative and corresponds to sequential execution in reading order (RTL fold).

A **phrase** is the ordered accumulation of word results:

```
P(phrase) = W1 ⊕ W2 ⊕ ... ⊕ Wk
```

Where `⊕` is ordered add = append to a list/sequence.

The VM maintains `A` as the phrase accumulator (ordered list of sealed word values).

### Word sealing

Define a single export function:

```
seal_word(state) -> WordValue
```

`WordValue` is an opaque, profile-defined value representing the sealed output of a word.

Minimum rule:

- If the current word has an explicit “last sealed artifact handle,” export that.
- Else export the current focus `F`.
- If the word is empty, export `⊥` (or the profile’s canonical empty value).

## Space semantics (`□`)

Whitespace tokens execute the **space operator** and are also injected implicitly at the beginning and end of input.

Space behavior is boundary-mode specific:

1. Advance logical time (`τ`):
   - `hard`, `glue`, `glue_maqqef`: `τ := τ + 1`
   - `cut`: `τ := τ + rank` (minimum rank `1`)
2. Seal/export current word:
   - `A := A ⊕ [ seal_word(state) ]`
3. Apply boundary resolution:
   - `glue` / `glue_maqqef`: create pending join and keep segment open.
   - `hard`: resolve pending obligations with default boundary policy.
   - `cut`: resolve pending obligations with strict cut policy; for `rank >= 3`, drop pending joins and flush unresolved mem-zone spill state.
4. Boundary-scope reset:
   - `glue` / `glue_maqqef`: do not reset segment obligations.
   - non-glue (`hard`/`cut`): clear obligation stack (`OStack_word := []`) and reset baseline focus/frame state.
5. Emit a boundary telemetry event:
   - `{ type: "BOUNDARY", mode, beforeFocus, afterFocus, segmentIdBefore }`
6. Optional GC pass (implementation policy).

When GC runs, minimum safety contract:

- MUST preserve bootstrap handles `Ω` and `⊥`.
- MUST preserve handles reachable from active roots (`Ω`, `F`, `R`, `K`, `W`) and any implementation-declared phrase/obligation roots.
- MUST scrub removed-handle references from graph structures (`links`, `boundaries`, `rules`, `cont`) and registers/lists deterministically.

Reference runtime policy (current):

- GC runs after `hard` and `cut` boundary application.
- Reachability roots include phrase/obligation structures (`A`, phrase accumulators/chunks, `E`, `OStack_word`, boundary/rule targets, pending-join anchors, constituent word-value references).
- Only ephemeral alias-like handles are collected (for example `kind=alias` or `meta.ephemeral=true`).

End-of-input behaves exactly like `□` (commit the final word into `A`).

### Cut boundary ranks

`mode=cut` uses rank-sensitive behavior:

- `rank >= 2`: applies boundary reset semantics for focus/frame state.
- `rank >= 3` (sof-pasuk class): MUST drop pending joins and flush unresolved mem-zone spill state before continuing boundary resolution.

Verse-level snapshot/reset after sof-pasuk is defined by `spec/phase-a-verse-cleanup.md`.

### Maqaf (`־`) tied-word semantics

Maqaf is an explicit tied-word separator and maps to glue semantics (`glue_maqqef`):

- the current word value is sealed and appended to phrase tracking,
- a pending join is created with join strength `maqqef`,
- boundary continuation stays open (no hard phrase commit at this boundary).

## WordStart semantics

Before the first executable letter/operator of each word, the VM MUST run `beginWord` exactly once:

```
beginWord(vm, wordText, prevBoundaryMode, F0)
```

Inputs:

- `wordText`: normalized full word surface for the word being entered.
- `prevBoundaryMode`: mode of the immediately preceding boundary token (`hard`, `glue`, `glue_maqqef`, `cut`).
- `F0`: inbound focus after boundary handling.

`beginWord` required behavior:

1. Reset word-local runtime state.
2. Segment transition:
   - if `prevBoundaryMode` is `glue` or `glue_maqqef`, keep current segment/OStack;
   - otherwise start a fresh segment (`segmentId += 1`, `segment.OStack := []`, `OStack_word := segment.OStack`).
3. Allocate a fresh word-local baseline construct `C0` (owner=`word`, role=`baseline`, empty payload).
4. Set `activeConstruct := C0` and `F := C0`.
5. Add symmetric alias relation `Alias(F0, C0)`.
6. Emit word-start telemetry:
   - `{ type: "WORD_START", wordText, prevBoundaryMode, F0, C0, segmentIdAfter, segmentReset }`

Runtime safety requirement:

- Letter/operator code MUST NOT perform baseline bootstrap fallback allocation (for example `if no construct then alloc`).
- If letter execution is reached without `activeConstruct`, runtime MUST fail early with a clear error.

## Interpreter output contract

The phrase interpreter returns:

- `A` as the phrase value (ordered list of `WordValue`).
- Optional but strongly recommended: `trace[i]`, the per-letter trace that produced `A[i]`.

Any higher-level meaning should start from `A[i]` values. Phrase meaning is composition over the list, not implicit cross-word binding.

## Obligations

Obligations are segment-scoped and must be discharged in LIFO order.

Segment rule:

- `glue` / `glue_maqqef` boundaries carry obligations forward into the next word in the same segment.
- non-glue boundaries (`hard`/`cut`) terminate the segment obligation scope before the next word begins.

```
OKind := { MEM_ZONE, SUPPORT } plus any additional kinds declared in `/registry/letters.yaml`.

Obligation := {
  kind: OKind,
  parent: handle_id,
  child: handle_id,
  payload: map,
  tau_created: int
}
```

Boundary defaults (required):

- hard/default boundary mode:
  - `MEM_ZONE` → close silently; no export.
  - `SUPPORT` → **fall**: log `fall(child,parent,τ)`, set `R := child`, `F := parent`.
- cut boundary mode:
  - `MEM_ZONE` → strict unresolved handling (spill/debt form, no silent close).
  - `SUPPORT` → strict unresolved handling (debt form, no automatic fall).

Any additional obligation kinds MUST declare their boundary defaults in `/registry/letters.yaml` and in letter specs.

Non-LIFO discharge is a compile error.

## Deterministic error handling

Invalid inputs MUST fail with typed errors:

- Compile-time errors (tokenization/validation)
- Runtime errors (VM/semantics)

No silent fallback for unknown diacritics or invalid nesting.
